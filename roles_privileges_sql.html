SQL Roles & Privileges

← Back to Dashboard

Roles and privileges in SQL define what actions a user can perform on the database. They are essential for securing data and managing user access control.

🔐 Common Privileges

  • SELECT — read data from tables/views
  • INSERT — add new data
  • UPDATE — modify existing data
  • DELETE — remove data
  • EXECUTE — run stored procedures/functions

🧑‍💼 Granting Privileges

GRANT SELECT, INSERT ON employees TO user_name;

Allows a user to view and add data to the employees table.

❌ Revoking Privileges

REVOKE INSERT ON employees FROM user_name;

Removes the ability to insert data into the table.

📦 Creating Roles

CREATE ROLE reporting_user;
GRANT SELECT ON reports TO reporting_user;
GRANT reporting_user TO user_name;

Roles allow grouping of privileges for easier management.

🧠 Tips & Best Practices

  • Grant only necessary privileges (principle of least privilege)
  • Use roles instead of managing individual permissions
  • Regularly audit roles and privileges
  • Document user access policies
