DML - INSERT, UPDATE, DELETE

← Back to Dashboard

DML (Data Manipulation Language) includes SQL commands used to manipulate data in existing database objects. Unlike DDL, which defines the structure, DML is all about modifying the actual content inside tables.

üì• INSERT Statement

The INSERT statement adds new records into a table. 

-- Insert one row
INSERT INTO employees (employee_id, full_name, department)
VALUES (101, 'Alice Johnson', 'HR');

-- Insert multiple rows
INSERT INTO employees (employee_id, full_name, department)
VALUES 
  (102, 'Bob Smith', 'Finance'),
  (103, 'Carol Lee', 'IT');

This adds one or more new records into the employees table.

‚úèÔ∏è UPDATE Statement

The UPDATE statement modifies existing records in a table. 

-- Update a single row
UPDATE employees
SET department = 'Marketing'
WHERE employee_id = 101;

-- Update multiple rows
UPDATE employees
SET department = 'General'
WHERE department = 'HR';

Always include a WHERE clause to avoid updating all records unintentionally.

üóëÔ∏è DELETE Statement

The DELETE statement removes one or more records from a table. 

-- Delete a specific employee
DELETE FROM employees
WHERE employee_id = 103;

-- Delete all employees in a department
DELETE FROM employees
WHERE department = 'Finance';

Use caution! DELETE permanently removes rows unless wrapped in a transaction.

üîê Transactions and Rollback

DML operations can be grouped into transactions. This allows you to ROLLBACK changes if something goes wrong. 

BEGIN;
UPDATE employees SET department = 'Unknown';
ROLLBACK;  -- Undo the change

-- Or use COMMIT to finalize changes
COMMIT;

üöÄ Real-World Uses

  • Adding new customer orders into an orders table
  • Updating product prices during a sale
  • Deleting expired sessions or logs
  • Building admin dashboards that modify backend data
