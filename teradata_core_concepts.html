üìê Teradata Architecture

Teradata is a massively parallel processing (MPP) RDBMS. Its architecture is based on shared-nothing systems where each unit of processing is independent. The core components include:

  • Parsing Engine (PE): Responsible for parsing queries, validating syntax, and generating execution plans.
  • AMPs (Access Module Processors): Each AMP handles a portion of the data and executes SQL steps in parallel.
  • BYNET: High-speed communication network that connects AMPs and PEs.
  • Disk Storage: Each AMP has its own dedicated disk storage, enabling parallel I/O operations.

üîë Primary and Secondary Indexes

Indexes determine how rows are distributed across AMPs:

  • Primary Index (PI): Determines row distribution using a hash function. Unique PI ensures even data distribution.
  • Secondary Index (SI): Provides alternate access paths but requires maintenance overhead.
  • Partitioned Primary Index (PPI): Improves query performance by reducing the number of rows scanned.

üìä Data Distribution and Skew

Teradata uses a hashing algorithm on the Primary Index to distribute rows across AMPs. A good PI ensures even distribution. Poor choice of PI can cause data skew, where some AMPs hold more data, affecting performance. Use Help Stats or DBQL to analyze skew factor.

üß† Teradata SQL Engine

Teradata supports ANSI SQL with enhancements for scalability and performance. The SQL engine coordinates with PEs and AMPs to execute complex queries in parallel, efficiently using resources.
