Informatica Sessions and Worklets

Learn how sessions and worklets execute and organize ETL logic in Informatica PowerCenter

← Back to Dashboard

📋 What is a Session?

A session is a task that executes a mapping. It defines how data is read from source, processed, and written to targets using session-level configuration.

⚙️ Session Configuration

Sessions can be configured with the following settings:

  • Connections (source/target)
  • Error handling and logging
  • Pre- and post-session commands
  • Parameter file and variable assignments

🔀 What is a Worklet?

A worklet is a reusable group of tasks (sessions, emails, commands, etc.) that can be reused in multiple workflows to promote modular design.

📦 Types of Worklets

  • Reusable Worklet: Can be used across multiple workflows.
  • Non-reusable Worklet: Created within a workflow and used only there.

🔁 Execution Order

Workflow Manager defines the order of execution using links and conditions (success, failure, skipped, etc.) between sessions and worklets.

🧰 Pre/Post-Session Commands

Shell commands or batch scripts can be configured to run before or after the session to handle file operations or notifications.

Pre-session command example:
mkdir -p /archive/input/

Post-session command example:
mv /input/file.csv /archive/input/

📈 Monitoring Sessions

Use Workflow Monitor to track session progress, view logs, and identify row-level read/write statistics or transformation errors.
