Top 100 SQL Interview Questions & Answers

Covers basic, intermediate, and advanced SQL concepts

← Back to Dashboard

1. What is SQL?

SQL (Structured Query Language) is used to communicate with databases. It allows you to retrieve, manipulate, insert, and delete data.

2. What are the different types of SQL statements?

DML (Data Manipulation), DDL (Data Definition), DCL (Data Control), and TCL (Transaction Control).

3. What is the difference between WHERE and HAVING?

WHERE filters rows before grouping, HAVING filters groups after aggregation.

4. What is a JOIN? Explain types of joins.

A JOIN combines rows from two or more tables. Types: INNER, LEFT, RIGHT, FULL OUTER, CROSS.

5. Write a query to fetch the second highest salary from a table.

SELECT MAX(salary)
FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);

6. What is the difference between DELETE, TRUNCATE, and DROP?

DELETE removes rows with a WHERE clause and can be rolled back. TRUNCATE removes all rows and cannot be rolled back. DROP deletes the table structure.

7. What is a primary key?

A primary key uniquely identifies each row in a table and cannot be NULL.

8. What is a foreign key?

A foreign key links records between two tables, referencing the primary key in another table.

9. What is normalization?

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

10. What is denormalization?

Denormalization adds redundancy to improve read performance by joining fewer tables.

11. What is the difference between CHAR and VARCHAR?

CHAR is fixed-length, VARCHAR is variable-length. VARCHAR saves space for varying-length values.

12. How do you find duplicate records in a table?

SELECT name, COUNT(*)
FROM employees
GROUP BY name
HAVING COUNT(*) > 1;

13. What is a subquery?

A subquery is a query nested within another SQL query. It is used to retrieve values to be used by the outer query.

14. What is a correlated subquery?

A correlated subquery depends on values from the outer query and runs once for each row processed by the outer query.

15. What is the use of COALESCE()?

COALESCE returns the first non-null value from the list of arguments.

SELECT COALESCE(NULL, NULL, 'default', 'backup'); -- returns 'default'

16. What does the GROUP BY clause do?

It groups rows that have the same values into summary rows, like totals or counts.

17. What is the difference between INNER JOIN and OUTER JOIN?

INNER JOIN returns only matching rows. OUTER JOIN returns all rows from one or both tables, with NULLs where there are no matches.

18. What is a CASE statement?

It is used to apply conditional logic in a SQL query.

SELECT name, 
  CASE 
    WHEN salary > 50000 THEN 'High'
    ELSE 'Low'
  END AS salary_level
FROM employees;

19. What is the difference between UNION and UNION ALL?

UNION removes duplicates; UNION ALL includes duplicates and is faster.

20. What is an alias in SQL?

An alias is a temporary name for a table or column, used for readability or brevity.

SELECT name AS employee_name FROM employees;

21. What does DISTINCT do in a SELECT statement?

DISTINCT removes duplicate records from the result set.

22. What is an index in SQL?

An index improves the speed of data retrieval operations on a database table.

23. What is the default sorting order in SQL?

Ascending (ASC) is the default sort order in SQL.

24. How do you sort records in descending order?

SELECT * FROM employees ORDER BY salary DESC;

25. How do you get the number of records in a table?

SELECT COUNT(*) FROM employees;

26. What does IS NULL do?

It checks for NULL values in a column.

SELECT * FROM employees WHERE manager_id IS NULL;

27. What is a self join?

A self join joins a table with itself to compare rows within the same table.

28. What is the LIMIT clause?

LIMIT restricts the number of records returned by a query (MySQL/PostgreSQL).

29. How to update multiple columns in SQL?

UPDATE employees
SET name = 'John Doe', salary = 60000
WHERE id = 101;

30. What is a view in SQL?

A view is a virtual table based on the result set of a SQL query. It does not store data itself.

31. What is a window function?

A window function performs a calculation across a set of rows related to the current row, without collapsing them into a group.

SELECT name, salary, RANK() OVER (ORDER BY salary DESC) AS rank FROM employees;

32. What is the difference between RANK and DENSE_RANK?

RANK leaves gaps between ranking values if there are duplicates, while DENSE_RANK does not.

33. What is the purpose of PARTITION BY in SQL?

PARTITION BY divides result sets into partitions to perform window functions independently for each partition.

34. What is the difference between stored procedures and functions?

Stored procedures perform operations but may not return values, while functions return a value and can be used in queries.

35. What is a transaction in SQL?

A transaction is a sequence of SQL operations that are executed as a single unit to maintain data consistency.

36. What are ACID properties?

ACID stands for Atomicity, Consistency, Isolation, and Durability — key properties ensuring reliable transactions.

37. What is the use of COMMIT and ROLLBACK?

COMMIT saves changes made by a transaction, while ROLLBACK undoes all changes since the last COMMIT.

38. How do you handle errors in a SQL procedure?

Use TRY/CATCH (SQL Server) or DECLARE HANDLER (MySQL) to capture and handle errors.

39. What is a trigger?

A trigger is a special procedure that runs automatically in response to specific database events like INSERT, UPDATE, DELETE.

40. How can indexes improve performance?

Indexes reduce the number of disk reads during queries by allowing quick lookups, but they can slow down writes.

41. What is a clustered vs non-clustered index?

A clustered index defines the physical order of data, while a non-clustered index maintains a separate structure.

42. What does EXPLAIN do in SQL?

EXPLAIN provides the execution plan of a query, showing how tables are joined and indexes are used.

43. What is a temporary table?

A temporary table stores intermediate results and is deleted automatically at the end of a session.

44. What are common reasons for query slowness?

Missing indexes, large scans, poor joins, outdated stats, and unfiltered queries are common causes.

45. What is denormalization and when is it used?

Denormalization adds redundancy to improve read performance, especially in analytical systems.

46. What is a correlated subquery?

A subquery that references columns from the outer query and is executed once for each row considered by the outer query.

47. What is the difference between EXISTS and IN?

EXISTS returns true if the subquery returns any rows. IN checks if a value matches any in a list. EXISTS is often more efficient with correlated subqueries.

48. What are common aggregate functions in SQL?

COUNT(), SUM(), AVG(), MIN(), MAX()

49. What is the purpose of the HAVING clause?

HAVING is used to filter results after a GROUP BY clause is applied, unlike WHERE which filters before grouping.

50. What is a CTE (Common Table Expression)?

A CTE is a temporary result set defined within the execution scope of a single SQL statement using the WITH clause.

WITH DeptCount AS (
  SELECT department, COUNT(*) AS emp_count
  FROM employees
  GROUP BY department
)
SELECT * FROM DeptCount WHERE emp_count > 5;

51. What is a recursive CTE?

A recursive CTE is a CTE that references itself, often used for hierarchical or tree-structured data.

52. How can you find the nth highest salary?

SELECT DISTINCT salary
FROM employees e1
WHERE N-1 = (
  SELECT COUNT(DISTINCT salary)
  FROM employees e2
  WHERE e2.salary > e1.salary
);

53. What are surrogate keys?

Surrogate keys are system-generated identifiers (e.g., auto-increment IDs) used instead of natural business keys.

54. What is an execution plan and how do you read it?

An execution plan shows how the SQL engine will execute a query, detailing index usage, joins, and estimated costs.

55. How can you improve the performance of a slow query?

Indexing, avoiding SELECT *, filtering early with WHERE, reducing joins, and using EXPLAIN to diagnose inefficiencies.

56. What is database sharding?

Sharding is the practice of splitting a large database into smaller, faster, more manageable pieces called shards.

57. What are materialized views?

Materialized views store the result of a query physically and are refreshed periodically. They improve performance at the cost of storage.

58. How do you prevent SQL injection?

Use parameterized queries, stored procedures, and input validation to prevent SQL injection attacks.

59. What is the difference between CHAR and VARCHAR?

CHAR is fixed-length, while VARCHAR is variable-length. CHAR is faster for fixed-size data, VARCHAR saves space.

60. How does indexing affect DML operations?

Indexes speed up SELECTs but slow down INSERT, UPDATE, DELETE because they require maintaining index data.

61. What is a composite key?

A composite key is a combination of two or more columns used together as a primary key.

62. What are constraints in SQL?

Constraints are rules enforced on data columns such as NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY, and CHECK.

63. What is a NULL value in SQL?

NULL represents a missing or unknown value. It is not the same as an empty string or zero.

64. What is the difference between DELETE and TRUNCATE?

DELETE removes rows one at a time and can be rolled back. TRUNCATE removes all rows and is faster, but can't be rolled back in many databases.

65. How do you rename a column in SQL?

-- MySQL
ALTER TABLE employees RENAME COLUMN old_name TO new_name;

66. What is a schema in SQL?

A schema is a logical container or namespace that holds database objects like tables, views, and procedures.

67. What is the use of COALESCE in SQL?

COALESCE returns the first non-null value in a list of expressions.

SELECT COALESCE(NULL, NULL, 'Fallback'); -- Returns 'Fallback'

68. What are SET operators in SQL?

SET operators combine the results of two SELECT queries. Common ones include UNION, UNION ALL, INTERSECT, and MINUS.

69. What is the use of the LIKE operator?

LIKE is used in a WHERE clause to search for a specified pattern in a column.

SELECT * FROM employees WHERE name LIKE 'A%';

70. What is the BETWEEN operator in SQL?

BETWEEN filters the result set within a certain range.

SELECT * FROM orders WHERE order_date BETWEEN '2023-01-01' AND '2023-12-31';

71. What is a data type in SQL?

A data type defines the kind of value a column can hold, such as INTEGER, VARCHAR, DATE, etc.

72. How can you get the current date in SQL?

SELECT CURRENT_DATE; -- PostgreSQL / Oracle
SELECT GETDATE();    -- SQL Server

73. What are SQL aliases used for?

Aliases rename columns or tables for display and readability.

SELECT salary AS monthly_salary FROM employees;

74. What is a CROSS JOIN?

A CROSS JOIN returns the Cartesian product of rows from the joined tables — every row from the first table combined with every row from the second.

75. How do you remove duplicate rows from a result?

Use SELECT DISTINCT to return unique rows only.

SELECT DISTINCT department FROM employees;

76. What is a UNION in SQL?

UNION combines the result of two or more SELECT statements, removing duplicates by default.

77. What is UNION ALL in SQL?

UNION ALL combines the result of two SELECT statements but includes duplicate records as well.

78. What is an OUTER JOIN?

An OUTER JOIN returns matching rows and also non-matching rows from either table with NULLs for missing values.

79. What is a RIGHT JOIN?

A RIGHT JOIN returns all records from the right table and the matched records from the left table.

80. What is a LEFT JOIN?

A LEFT JOIN returns all records from the left table and the matched records from the right table.

81. How do you retrieve records within a range?

Use the BETWEEN operator for filtering data within a specific range.

82. What is a CHECK constraint?

A CHECK constraint limits the values that can be entered into a column by validating a condition.

83. What is a default constraint?

It assigns a default value to a column when no value is specified during insert.

84. What are scalar functions?

Scalar functions return a single value based on input. Examples: UPPER(), LEN(), NOW().

85. How to rename a table in SQL?

ALTER TABLE old_table_name RENAME TO new_table_name;

86. How do you calculate the length of a string in SQL?

Use LENGTH() in MySQL, LEN() in SQL Server.

87. What is the purpose of the NVL() function?

In Oracle, NVL() replaces NULL with a specified value.

SELECT NVL(salary, 0) FROM employees;

88. What are system functions?

Functions that return system-level information, like CURRENT_DATE, USER, SESSION_USER.

89. What is the difference between NOW() and CURRENT_TIMESTAMP?

They often return the same value (current datetime), but syntax varies slightly across RDBMSs.

90. What are built-in functions in SQL?

SQL has many built-in functions including string, numeric, date/time, and aggregate functions.

91. What is a synonym in SQL?

A synonym is an alias or alternative name for a database object such as a table or view, often used for abstraction or cross-schema access.

92. What is a database cursor?

A cursor is a database object used to retrieve, manipulate, and navigate row-by-row results of a query, typically in stored procedures.

93. What is the difference between logical and physical data independence?

Logical independence is the ability to change the logical schema without affecting applications; physical independence refers to changes in storage structure without impacting the schema.

94. What are the differences between OLTP and OLAP databases?

OLTP (Online Transaction Processing) is optimized for write operations and quick lookups. OLAP (Online Analytical Processing) is designed for complex queries and analytics.

95. What is a surrogate key?

A surrogate key is a unique identifier for an entity, typically a sequential number generated by the system, with no business meaning.

96. How can you list all table names in a SQL database?

Use catalog views or information schema, e.g.:

SELECT table_name FROM information_schema.tables WHERE table_type = 'BASE TABLE';

97. What is the purpose of ROW_NUMBER() in SQL?

ROW_NUMBER() assigns a unique sequential integer to rows within a partition of a result set, often used for pagination.

98. What is the difference between DELETE and DROP?

DELETE removes data from a table but retains the table structure. DROP deletes the table and its data permanently.

99. How can you find the highest value in a column?

SELECT MAX(salary) FROM employees;

100. What is the difference between a primary key and a unique key?

Both enforce uniqueness, but a table can have only one primary key (which cannot contain NULLs) and multiple unique keys (which can include NULLs).
