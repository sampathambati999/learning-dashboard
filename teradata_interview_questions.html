Teradata Interview Questions

A collection of common Teradata interview questions with brief answers to help you prepare effectively.

1. What is Teradata and what makes it unique?

Teradata is an enterprise-class RDBMS known for its massively parallel processing (MPP) architecture. It can handle huge volumes of data and supports linear scalability and high-performance analytics.

2. What is a Primary Index in Teradata?

A Primary Index determines how data is distributed across AMPs using a hashing algorithm. It plays a crucial role in data retrieval and query performance.

3. Explain the difference between Unique and Non-Unique Primary Index.

A Unique PI ensures even data distribution and avoids duplicates. A Non-Unique PI may cause uneven data spread and increase skew if too many rows share the same PI value.

4. What is a Skew Factor and how do you handle it?

Skew Factor measures the uneven distribution of rows across AMPs. A high skew can degrade performance. It can be resolved by selecting a better Primary Index or partitioning strategy.

5. What is the difference between FastLoad and MultiLoad?

FastLoad is used to insert data into empty tables only and does not allow updates. MultiLoad supports INSERT, UPDATE, DELETE operations and can work with populated tables.

6. How does Teradata handle locking?

Teradata automatically locks tables/rows depending on the SQL operation. It supports Access, Read, Write, and Exclusive locks to ensure data consistency.

7. What is Collect Statistics and why is it important?

Collecting statistics provides the optimizer with information about data demographics, which is crucial for generating efficient execution plans.

8. Explain the usage of QUALIFY clause.

QUALIFY allows filtering rows after analytical functions like RANK or ROW_NUMBER are applied, avoiding subqueries.

10. How do you export data from Teradata?

You can use the EXPORT utility, BTEQ, or TPT scripts to export data into flat files. Export operations support various formats and can be run in parallel for performance.
