üì¶ Stored Procedures & Macros

Stored Procedures are reusable blocks of SQL that include logic, variables, loops, and conditionals. They allow automation of multi-step operations and return outputs like cursors or status messages.

Macros are precompiled SQL commands that can be executed with different parameters. They are more lightweight but do not support procedural logic like loops or IF-ELSE conditions.

Teradata supports various window and aggregation functions such as:

  • RANK(), DENSE_RANK(), ROW_NUMBER()
  • QUALIFY clause to filter windowed results
  • COALESCE() for null handling
  • LISTAGG() for string aggregation
  • CASE WHEN and conditional logic

‚ùå Error Handling

Teradata utilities like FastLoad and MultiLoad generate error tables during the load process. Logs capture duplicate rows, constraint violations, and transformation errors. Use SHOW TABLE and diagnostic SQL to review error tables and resolve data issues.

üßæ Soft Deletes

Instead of physically deleting records, soft deletes update a status column (e.g., is_deleted = 'Y') to mark data as logically removed. This maintains history, aids recovery, and improves auditability. Be sure to filter soft-deleted rows in reports using WHERE clauses.
