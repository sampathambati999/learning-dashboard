DDL - CREATE, ALTER, DROP

← Back to Dashboard

DDL (Data Definition Language) is a subset of SQL used to define and manage all structures in a database, such as tables, schemas, views, indexes, and more. These statements define how data is stored and organized.

üìå CREATE Statement

The CREATE statement is used to create new database objects like tables, indexes, views, or entire databases. 

CREATE TABLE students (
  student_id INT PRIMARY KEY,
  name VARCHAR(100),
  birth_date DATE
);

This creates a table named students with three columns.

üîß ALTER Statement

The ALTER statement modifies the structure of an existing database object, such as adding, deleting, or modifying columns in a table. 

-- Add a new column
ALTER TABLE students ADD email VARCHAR(255);

-- Modify a column's data type
ALTER TABLE students MODIFY name VARCHAR(150);

-- Drop a column
ALTER TABLE students DROP COLUMN birth_date;

These commands change the structure of the students table dynamically.

üóëÔ∏è DROP Statement

The DROP statement is used to delete existing database objects. Once dropped, the data and structure are permanently removed. 

DROP TABLE students;

This command deletes the students table completely from the database.

‚ö†Ô∏è Important Notes

  • DDL operations are auto-committed. Once executed, they cannot be rolled back unless explicitly wrapped in a transaction block (depending on the DBMS).
  • Always be cautious when using DROP ‚Äî it removes data permanently.
  • ALTER syntax may vary slightly between database systems (MySQL, Oracle, PostgreSQL).

üöÄ Use Cases

DDL commands are typically used during the initial design of a database, as well as when evolving the schema to meet new business or data requirements.
