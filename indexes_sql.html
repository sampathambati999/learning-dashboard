SQL Indexes

← Back to Dashboard

An index in SQL is a performance-tuning method of allowing faster retrieval of records from a table. Indexes are especially useful for large tables and frequent queries.

üìå Creating an Index

CREATE INDEX idx_employees_department
ON employees (department);

Creates an index on the department column to speed up queries that filter or sort by department.

üîç Using Indexes

Indexes are automatically used by the SQL engine when appropriate. You don't need to change your SELECT queries to use them.

üßπ Dropping an Index

DROP INDEX idx_employees_department;

Removes the index. Syntax may vary by database system.

üß† Types of Indexes

  • Single-column index: One column (e.g., department)
  • Composite index: Multiple columns (e.g., last_name, first_name)
  • Unique index: Ensures no duplicate values
  • Full-text index: For searching textual data (e.g., articles)
  • Clustered index: Defines the physical order of data (one per table)
  • Non-clustered index: Stored separately; most common type
