Versioning & Repository Management

Track changes, manage collaboration, and ensure governance using Informatica’s versioning and repository tools

← Back to Dashboard

📁 What is a Repository?

An Informatica repository is a database that stores all metadata for mappings, transformations, workflows, and other objects.

🔍 Version Control System

Informatica provides built-in version control features to help manage changes over time.

  • Object check-in/check-out
  • Version history tracking
  • Rollback to previous versions
  • Labeling and commenting versions

🔐 Security and Locking

When a developer checks out an object, it becomes locked to others to prevent overwrites. Only the owner can check it back in.

🛠️ Object History and Comparison

You can compare versions of a mapping to identify changes or restore earlier configurations.

Repository Manager → Right-click → View History → Compare Versions

🧱 Repository Types

  • Global Repository: Shared across multiple domains
  • Local Repository: Belongs to a single domain
  • Standalone Repository: Not connected to other repositories

✅ Best Practices

  • Always check-in objects with meaningful comments
  • Use labels/tags to track releases
  • Schedule repository backups regularly
  • Maintain user roles to enforce access control
