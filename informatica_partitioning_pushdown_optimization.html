Partitioning divides data into segments that are processed in parallel, increasing throughput and reducing session runtime.

  • Informatica Pipeline Partitioning: Defines partitions at source/target level or in transformations
  • Fixed Partitioning: Manually sets the number of partitions
  • Key Range Partitioning: Based on a range of values (e.g., date range, region ID)

üìâ What is Pushdown Optimization?

Pushdown Optimization allows Informatica to translate transformations into SQL and push execution to the database.

üéØ Levels of Pushdown Optimization

  • Source-level: Only push filters/joins at the source
  • Full Pushdown: Most transformations are pushed to DB (if supported)
  • Target-level: Execute DML directly in target DB
