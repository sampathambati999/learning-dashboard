Informatica Transformations Overview

A guide to essential active and passive transformations in Informatica PowerCenter

← Back to Dashboard

🔁 Passive vs Active Transformations

Passive transformations do not change the number of rows passing through them, while active transformations can increase, decrease, or reorder rows.

🧮 Expression Transformation (Passive)

Used to calculate values in a single row using expressions, string manipulation, conditional logic, etc.

IIF(SALARY > 10000, 'High', 'Low')

🚪 Filter Transformation (Active)

Filters rows based on a condition. Only matching rows are passed through.

DEPARTMENT = 'HR'

🔗 Joiner Transformation (Active)

Used to join two heterogeneous sources (e.g., flat file + database). Supports Normal, Master Outer, Detail Outer, and Full Outer joins.

📦 Lookup Transformation (Passive/Active)

Performs lookup on a relational table, flat file, or pipeline to fetch related data. Can be connected or unconnected.

📊 Aggregator Transformation (Active)

Performs aggregate calculations such as SUM, COUNT, AVG, MAX, MIN grouped by specified columns.

SUM(SALARY) GROUP BY DEPARTMENT

🚦 Router Transformation (Active)

Routes data into multiple groups based on conditions, similar to a Filter but supports multiple output groups.

📃 Sorter Transformation (Active)

Sorts rows based on one or more ports (columns). It is active because it can change the order and block data until sort completes.

🧵 Sequence Generator (Passive)

Generates unique numeric values, often used to populate surrogate keys.

💡 Union Transformation (Active)

Merges data from multiple pipelines into a single pipeline. All input ports must be the same structure.

🧼 Update Strategy (Active)

Determines whether rows should be inserted, updated, deleted, or rejected based on business logic.

IIF(NEW_FLAG = 'Y', DD_INSERT, DD_UPDATE)
