Teradata Administration & Security

An overview of essential administrative tasks and access control mechanisms in Teradata.

üë§ User and Role Management

Teradata supports multi-level security using users, roles, and profiles. Users can be assigned to roles to simplify privilege management. System administrators use the CREATE USER, CREATE ROLE, and GRANT statements to manage access.

CREATE ROLE FinanceRole;
GRANT SELECT, INSERT ON SalesDB TO FinanceRole;
GRANT FinanceRole TO user1;

üîê Access Rights

Access rights define what users can do with database objects. Common privileges include:

  • SELECT, INSERT, UPDATE, DELETE
  • EXECUTE (for procedures)
  • CREATE, DROP, ALTER
  • WITH GRANT OPTION to pass privileges to others

üìã Profiles

Profiles are used to assign default settings such as spool space, password policies, and session limits to multiple users. This simplifies user administration and enforces consistency.

CREATE PROFILE AnalystProfile
  AS SPOOL = 5000000
     PASSWORD = (EXPIRY = 90);
CREATE USER analyst1 FROM AnalystProfile;

üîí Locking and Concurrency Control

Teradata uses automatic locking to ensure data consistency. Lock types include:

  • Read Lock: Allows others to read; prevents updates
  • Write Lock: Prevents all other access
  • Access Lock: Used for non-critical reads without blocking

Use SHOW LOCKS or DBQL to diagnose lock contention.

üìë Auditing and Query Logging

The DBQL (Database Query Log) captures information about SQL requests, user activity, execution time, and more. This helps with tuning and auditing user behavior.

  • Enable DBQL per user or globally
  • Capture query text, CPU usage, I/O usage
  • Analyze usage for optimization or auditing
